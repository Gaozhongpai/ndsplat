\begin{abstract}
   High-dimensional Gaussian Splatting (N-DGS) methods like 6DGS and 7DGS achieve impressive view- and temporal-dependent modeling through conditional N-dimensional Gaussians. However, their monolithic covariance design derives all conditioning effects from a joint covariance matrix, requiring matrix inversions, regression multiplications, and covariance corrections per Gaussian in every forward pass. We introduce \modelname{}, which replaces this implicit derivation with direct parameterization: opacity modulation via Cholesky precision factors and position shifts via displacement matrices with learnable per-dimension coupling strengths. This eliminates matrix inversions and covariance corrections, achieving 5--6$\times$ faster slicing kernels and up to 2.66$\times$ faster end-to-end rendering. Across five benchmarks spanning static, dynamic, and specular scenes, \modelname{} consistently matches or exceeds N-DGS quality on every dataset (up to +0.97 dB on Mip-NeRF 360), offering a principled, efficient alternative to covariance-derived conditioning in high-dimensional Gaussian splatting.
\end{abstract}

\section{Introduction}
\label{sec:introduction}
3D Gaussian Splatting (3DGS)~\citep{3dgs} revolutionized novel-view synthesis by achieving photorealistic quality at real-time speeds through explicit 3D Gaussian primitives. However, real-world scenes exhibit complex phenomena beyond static representations: specular reflections, temporal dynamics, and view-dependent effects that require richer modeling capacity.

Recent extensions elevate Gaussians into higher-dimensional spaces to capture these phenomena. N-dimensional Gaussians (N-DG)~\citep{diolatzis2024n} pioneered this direction, refined by 6D Gaussian Splatting (6DGS)~\citep{gao20246dgs} which models position and viewing direction jointly for view-dependent effects, and 7D Gaussian Splatting (7DGS)~\citep{gao20257dgs} which adds temporal dimensions for dynamic scenes (collectively referred to as N-DGS). Alternative kernel designs like Deformable Beta Splatting (DBS)~\citep{liu2025deformable} improve geometric fidelity but remain limited to spatial dimensions.


Despite their expressive power, these N-dimensional formulations suffer from a fundamental limitation: their covariance-based parameterization. All conditioning effects (position shifts, opacity modulation, and covariance adjustments) are derived from a single joint covariance matrix via conditional Gaussian formulas. Computing any single effect requires inverting the conditioning block, performing regression matrix multiplications, and computing covariance corrections per Gaussian, creating computational overhead that accumulates across millions of primitives. Moreover, this implicit derivation offers limited control: position and opacity conditioning are rigidly tied through the shared covariance structure, preventing independent adjustment of their coupling strengths. This is problematic because different phenomena require different conditioning strategies. For instance, view-dependent reflections primarily need opacity modulation, while temporal motion requires position displacement, yet N-DGS's fixed coupling forces the same relationship for all Gaussians. The monolithic design also risks numerical instabilities when covariance matrices become ill-conditioned.

\textbf{Our Insight.} Conditional Gaussian formulas are elegant mathematically but computationally inefficient. Each conditioning effect can be \textbf{directly parameterized} with efficient, independent parameters instead of being implicitly derived through the covariance matrix. Opacity modulation only needs a precision matrix for distance computation; therefore, we parameterize it directly via Cholesky factors. Position shifts only need a displacement matrix scaled by query deviations; therefore, we parameterize it directly with bounded weights. By replacing implicit derivation with explicit parameters, we eliminate matrix inversions while gaining fine-grained control over each effect independently.

We introduce \textbf{\modelname{}}, a direct parameterization for efficient conditional Gaussian splatting. The `\textbf{d}' signifies our principle: \textbf{d}irect parameterization---explicit \textbf{d}isplacement matrices for position and \textbf{d}ecay factors for opacity, enabling \textbf{d}ynamic Gaussian behavior. Like N-DGS, we enable Gaussians to adaptively shift position and modulate opacity based on queries, but replace implicit covariance derivation with efficient explicit parameters. Our key contributions are:


\begin{itemize}[leftmargin=*]
\item \textbf{Direct Opacity Parameterization:} We directly parameterize opacity decay via Cholesky precision matrices, eliminating the need for matrix inversions while guaranteeing numerical stability through triangular operations.

\item \textbf{Direct Position Displacement:} Position shifts are directly parameterized via displacement matrices with spatial scaling and bounded activations, ensuring tight displacement bounds without requiring covariance inversion.

\item \textbf{Learnable Coupling Strength:} Per-dimension coupling parameters control how strongly position shifts respond to query deviations, enabling adaptive interpolation from no position conditioning to full coupling through the shared precision matrix.

\item \textbf{Efficient CUDA Implementation:} We implement optimized CUDA kernels for both our method and N-DGS baseline, fusing conditional operations with register-level computation for fair comparison (Note that the original N-DGS only implements in PyTorch).
\end{itemize}

Experiments demonstrate that \modelname{} matches or exceeds N-DGS quality across all benchmarks while achieving up to 2.66$\times$ faster rendering (166\% throughput increase) on in-the-wild scenes (Mip-NeRF 360).


\section{Related Work}
\label{sec:related_work}

\textbf{3D Gaussian Splatting.}
3D Gaussian Splatting (3DGS)~\citep{3dgs} achieves real-time novel view synthesis through explicit 3D Gaussian primitives with spherical harmonic color encoding. This has inspired extensions addressing multi-scale anti-aliasing~\citep{yu2023mipsplattingaliasfree3dgaussian,yan2024multiscale3dgaussiansplatting}, compression~\cite{lee2024compact,morgenstern2024compact}, and semantic modeling~\cite{shi2023languageembedded3dgaussians,zhou2024feature3dgssupercharging3d}. However, 3DGS remains limited to static geometry, relying on spherical harmonics for view-dependent appearance without mechanisms for temporal dynamics or view-dependent geometry.

\textbf{Alternative Kernel Designs.}
Several works explore non-Gaussian kernels to overcome limitations in 3D spatial modeling. Modified Gaussian variants include generalized exponentials~\cite{hamdi2024gesgeneralizedexponentialsplatting}, truncated kernels~\cite{liu2025tnt}, second-order formulations~\cite{zhang2025quadratic}, half-Gaussians~\cite{li20253d}, and discontinuity-aware profiles~\cite{qu2024disc}. Alternative probability distributions have also been explored: Beta kernels~\cite{liu2025deformable} for improved geometric fidelity, Student-t distributions~\cite{zhu20253dstudentsplattingscooping} for heavier tails, and Gabor functions~\cite{zhou20253dgabsplat3dgaborsplatting} for sharper features. Beyond volumetric representations, methods using convex shapes~\cite{held20243dconvexsplattingradiance}, triangles~\cite{held2025triangle}, and radial kernels~\cite{huang2025deformable} offer alternative geometric primitives. However, all these approaches focus on 3D spatial kernels and do not extend to higher dimensions for modeling view-dependent or temporal effects.

\textbf{Higher-Dimensional Gaussian Splatting.}
To model view-dependent and temporal effects, recent methods lift Gaussians into higher-dimensional spaces. 4DGS~\cite{yang2023real} extends 3DGS with temporal dimensions for dynamic scene reconstruction. N-DG~\cite{diolatzis2024n} establishes a general framework for arbitrary-dimensional Gaussians. Building on this, 6DGS~\cite{gao20246dgs} jointly models spatial position and viewing direction for view-dependent rendering, while 7DGS~\cite{gao20257dgs} unifies spatial, angular, and temporal dimensions. We refer to 6DGS and 7DGS collectively as N-DGS. UBS~\citep{liu2025universal} extends this paradigm by applying Beta kernels to N-dimensional representations. A key limitation is that N-DGS methods derive all conditioning effects from joint covariance matrices, requiring matrix inversions, regression multiplications, and covariance corrections per primitive, adding significant per-Gaussian overhead and risking numerical instability.

\textbf{Dynamic Scene Modeling.}
NeRF-based methods model dynamics through deformations: D-NeRF~\cite{pumarola2020dnerfneuralradiancefields} introduced canonical-to-deformed mappings, Nerfies~\cite{park2021nerfies} handled non-rigid motion, and HyperNeRF~\cite{park2021hypernerf} captured topological changes. For Gaussian Splatting, 4DGS~\cite{yang2023real} enables real-time dynamic rendering, while Dynamic 3D Gaussians~\cite{luiten2023dynamic} combines canonical representations with learned deformations. FreeTimeGS~\cite{wang2025freetimegs} allows Gaussians to appear at arbitrary times and locations with motion functions. These deformation-based approaches model dynamics as transformations of primitives, whereas 7DGS~\cite{gao20257dgs} achieves unified modeling through higher-dimensional conditioning.


\section{Preliminary}

\paragraph{3D Gaussian Splatting}
3D Gaussian Splatting (3DGS)~\cite{3dgs} revolutionized novel view synthesis by representing scenes as a collection of anisotropic 3D Gaussians. Each Gaussian is a primitive defined by a set of learnable attributes that describe its geometry and appearance. Its geometry is determined by a mean vector \(\boldsymbol{\mu} \in \mathbb{R}^3\) specifying its center in 3D space, and a covariance matrix \(\boldsymbol{\Sigma} \in \mathbb{R}^{3 \times 3}\) defining its shape, scale, and orientation. \(\boldsymbol{\Sigma}\) is parameterized by a diagonal scaling matrix \(\mathbf{S} = \operatorname{diag}(s_x, s_y, s_z)\) and a rotation matrix \(\mathbf{R}\) (represented as a quaternion), such that \(\boldsymbol{\Sigma} = \mathbf{R} \mathbf{S} \mathbf{S}^\top \mathbf{R}^\top\). The appearance of each Gaussian is governed by a scalar opacity \(\alpha \in [0, 1]\) that controls its transparency, and a view-dependent color modeled using Spherical Harmonics (SH). The color \(c\) for a given viewing direction \(d\) is computed as a sum over the SH basis functions \(Y_{\ell m}\):
\begin{equation}
c(d) = \sum_{\ell=0}^{L} \sum_{m=-\ell}^{\ell} \beta_{\ell m} Y_{\ell m}(d),
\end{equation}
where \(\beta_{\ell m}\) are the learnable SH coefficients. 3DGS achieves state-of-the-art quality and real-time rendering for static scenes. However, its representation is fundamentally static and cannot model geometric changes arising from complex view-dependent effects or temporal dynamics.

\paragraph{N-Dimensional Gaussian Splatting}

To capture view-dependent and temporal effects, recent methods extend 3DGS into higher dimensions: 6DGS~\cite{gao20246dgs} for view-dependent phenomena and 7DGS~\cite{gao20257dgs} for dynamic scenes. Each primitive is an N-dimensional Gaussian over position $\mathbf{X}_p \in \mathbb{R}^3$ and query variables $\mathbf{X}_q \in \mathbb{R}^C$ (view direction for 6DGS with $C=3$, view+time for 7DGS with $C=4$):
\begin{align}
\mathbf{X} = \begin{pmatrix} \mathbf{X}_p \\ \mathbf{X}_q \end{pmatrix} \sim \mathcal{N}\!\left(
\begin{pmatrix}
\boldsymbol{\mu}_p \\ \boldsymbol{\mu}_q
\end{pmatrix},
\begin{pmatrix}
\boldsymbol{\Sigma}_{pp} & \boldsymbol{\Sigma}_{pq} \\
\boldsymbol{\Sigma}_{pq}^\top & \boldsymbol{\Sigma}_{qq}
\end{pmatrix}
\right).
\end{align}
At query $\mathbf{q}$, conditioning yields a 3D Gaussian with parameters:
\begin{align}
\alpha_{\text{cond}} &= \alpha \cdot \exp\left(-\lambda\,(\mathbf{q}-\boldsymbol{\mu}_q)^\top \boldsymbol{\Sigma}_{qq}^{-1}(\mathbf{q}-\boldsymbol{\mu}_q)\right),\\
\boldsymbol{\mu}_{\text{cond}} &= \boldsymbol{\mu}_p + \boldsymbol{\Sigma}_{pq}\,\boldsymbol{\Sigma}_{qq}^{-1}(\mathbf{q}-\boldsymbol{\mu}_q), \\
\boldsymbol{\Sigma}_{\text{cond}} &= \boldsymbol{\Sigma}_{pp} - \boldsymbol{\Sigma}_{pq}\,\boldsymbol{\Sigma}_{qq}^{-1}\,\boldsymbol{\Sigma}_{pq}^\top,
\end{align}
where $\lambda$ is a scaling factor (default 0.35). This formulation requires inverting $\boldsymbol{\Sigma}_{qq}$, computing the regression matrix $\boldsymbol{\Sigma}_{pq}\boldsymbol{\Sigma}_{qq}^{-1}$, and evaluating the covariance correction $\boldsymbol{\Sigma}_{pq}\boldsymbol{\Sigma}_{qq}^{-1}\boldsymbol{\Sigma}_{pq}^\top$ for every Gaussian in every forward pass. While $C$ is small (3 or 4), these three matrix operations collectively add significant per-Gaussian overhead that accumulates across millions of primitives and risks numerical instability during training, motivating our simpler direct parameterization.

\section{Our Approach}

N-DGS methods derive all conditioning effects from a single $(3+C) \times (3+C)$ joint covariance matrix via conditional Gaussian formulas. To compute any effect (position shift, opacity modulation, or covariance adjustment), they must invert $\boldsymbol{\Sigma}_{qq}$, multiply regression matrices, and compute covariance corrections, requiring multiple matrix operations per Gaussian per forward pass.

We propose \modelname{}, which replaces this \textbf{implicit derivation} with \textbf{direct parameterization}. Rather than computing conditioning effects through matrix algebra, we explicitly parameterize each component with dedicated, efficient parameters. This direct approach eliminates matrix inversions and covariance corrections while providing independent control over each conditioning effect.

\subsection{Overview}

Our approach directly parameterizes conditional Gaussian slicing with two explicit components. Given the deviation $\boldsymbol{\delta} = q - \mu_q$ from the canonical view/time:
\begin{align}
\alpha_{\text{cond}} &= \alpha \cdot \exp\left(-\|\mathbf{L}^\top \boldsymbol{\delta}\|_2^2\right), & &\text{(Opacity)} \\
\mu_{\text{cond}} &= \mu_p + \mathbf{V}_{pq} \cdot \tilde{\mathbf{V}}_{qq} \cdot \boldsymbol{\delta}, & &\text{(Position)} \end{align}
where $\mathbf{L} \in \mathbb{R}^{C \times C}$ is the directly parameterized Cholesky factor, $\mathbf{V}_{qq} = \mathbf{L}\mathbf{L}^\top$ is the precision matrix computed directly from $\mathbf{L}$ without inversion, $\mathbf{V}_{pq} = \bar{s}\boldsymbol{\Theta}_{pq} \in \mathbb{R}^{3 \times C}$ is the displacement matrix (mean spatial scale $\bar{s}$ times normalized $\boldsymbol{\Theta}_{pq}$ with $\|\boldsymbol{\Theta}_{pq}\|_F = 1$), and $\tilde{\mathbf{V}}_{qq} = \text{diag}(\boldsymbol{\Lambda}) \mathbf{V}_{qq}$ is the scaled precision with learnable per-dimension coupling strengths $\boldsymbol{\Lambda} \in [0,1]^C$ (dimensions of the same type share parameters). The spatial covariance $\boldsymbol{\Sigma}_{pp}$ remains independent of the query, using the standard 3DGS quaternion-scale parameterization ($\boldsymbol{\Sigma}_{pp} = \mathbf{R}\mathbf{S}\mathbf{S}^\top \mathbf{R}^\top$) for seamless compatibility with existing pipelines, as densification strategies rely on the spatial covariance.

For dynamic scenes with view and time conditioning ($C=4$), we further \textbf{separate view and time} by using a block-diagonal precision matrix with zero cross-terms:
\begin{equation}
\mathbf{V}_{qq} = \mathbf{L}\mathbf{L}^\top = \begin{pmatrix}
\mathbf{V}_{vv} & \mathbf{0} \\
\mathbf{0} & v_t
\end{pmatrix},
\end{equation}
where $\mathbf{V}_{vv} \in \mathbb{R}^{3 \times 3}$ is the view precision and $v_t \in \mathbb{R}$ is the time precision. This separation eliminates coupling between spatial view directions and temporal dynamics, reflecting the physical intuition that view-dependent effects and temporal changes are independent phenomena.

N-DGS must compute the regression matrix $\mathbf{K} = \boldsymbol{\Sigma}_{pq}\boldsymbol{\Sigma}_{qq}^{-1}$, which is reused for position shift and covariance correction; \modelname{} eliminates all three operations. The key innovation is \textbf{learnable coupling strength} via per-dimension parameters $\Lambda$: when $\Lambda = \mathbf{0}$, position becomes query-independent ($\boldsymbol{\mu}_{\text{cond}} = \boldsymbol{\mu}_p$), which we denote as the simplified \textbf{\textit{d}GS-O} variant using only direct opacity parameterization for maximum efficiency; when $\Lambda = \mathbf{1}$, position couples through the shared precision matrix $\mathbf{V}_{qq}$ like N-DGS. The following subsections detail the direct parameterization of each component.

\subsection{Opacity Conditioning}

In N-DGS, opacity is modulated as $\alpha_{\text{cond}} = \alpha \cdot \exp(-\lambda\,(\mathbf{q}-\boldsymbol{\mu}_q)^\top \boldsymbol{\Sigma}_{qq}^{-1}(\mathbf{q}-\boldsymbol{\mu}_q))$, requiring matrix inversion.

Our formulation directly parameterizes the precision matrix via its Cholesky factor $\mathbf{L}$, computing $\mathbf{L}^\top \boldsymbol{\delta}$ followed by $\|\cdot\|_2^2$ through a triangular matrix-vector multiplication. This eliminates the need for matrix inversion. Parameterizing the Cholesky factor directly with exponential activation on the diagonal guarantees that $\mathbf{V}_{qq} = \mathbf{L}\mathbf{L}^\top$ is always symmetric positive definite, avoiding numerical instabilities from ill-conditioned matrices.

For dynamic scenes, the block-diagonal structure enables \textbf{separated view and time opacity}. With $\boldsymbol{\delta} = (\boldsymbol{\delta}_v, \delta_t)^\top$ where $\boldsymbol{\delta}_v \in \mathbb{R}^3$ is the view deviation and $\delta_t \in \mathbb{R}$ is the time deviation, the opacity factors as:
\begin{equation}
\alpha_{\text{cond}} = \alpha \cdot \exp\left(-\left(\boldsymbol{\delta}_v^\top \mathbf{V}_{vv} \boldsymbol{\delta}_v + \delta_t^2 v_t\right)\right).
\end{equation}
This factorization allows view- and time-dependent opacity to be controlled independently, with view precision $\mathbf{V}_{vv}$ parameterized by a $3 \times 3$ Cholesky factor (6 parameters) and time precision $v_t$ by a single scalar, for a total of 7 parameters instead of 10 for a full $4 \times 4$ precision matrix.

\subsection{Position Conditioning}
\label{subsec:position}

In N-DGS, position is computed as $\boldsymbol{\mu}_{\text{cond}} = \boldsymbol{\mu}_p + \boldsymbol{\Sigma}_{pq}\boldsymbol{\Sigma}_{qq}^{-1}(\mathbf{q}-\boldsymbol{\mu}_q)$, where both $\boldsymbol{\Sigma}_{pq}$ and $\boldsymbol{\Sigma}_{qq}^{-1}$ are implicitly derived from the joint covariance matrix through Cholesky decomposition and matrix inversion.

We replace this implicit derivation with \textbf{direct parameterization} of position shifts, combined with learnable coupling strength control:
\begin{equation}
\vspace{-0.3em}
\boldsymbol{\mu}_{\text{cond}} = \boldsymbol{\mu}_p + \underbrace{\bar{s} \boldsymbol{\Theta}_{pq}}_{\mathbf{V}_{pq}} \cdot \underbrace{\text{diag}(\Lambda) \mathbf{V}_{qq}}_{\tilde{\mathbf{V}}_{qq}} \cdot \boldsymbol{\delta},
\vspace{-0.3em}
\end{equation}
where $\bar{s} = \text{mean}(s_x, s_y, s_z)$ is the spatial scale, $\boldsymbol{\Theta}_{pq} \in \mathbb{R}^{3 \times C}$ are learnable parameters normalized to unit Frobenius norm ($\|\boldsymbol{\Theta}_{pq}\|_F = 1$), $\text{diag}(\boldsymbol{\Lambda}) \in \mathbb{R}^{C \times C}$ contains learnable per-dimension coupling strengths in $[0,1]$, and $\mathbf{V}_{qq} = \mathbf{L}\mathbf{L}^\top$ is the precision matrix shared with opacity conditioning. The coupling strength $\boldsymbol{\Lambda}$ controls how strongly position shifts respond to query deviations:
\vspace{-0.5em}
\begin{itemize}[leftmargin=*]
\item When $\Lambda = \mathbf{0}$: $\tilde{\mathbf{V}}_{qq} = \mathbf{0}$, position becomes query-independent ($\boldsymbol{\mu}_{\text{cond}} = \boldsymbol{\mu}_p$, no conditioning).
\item When $\Lambda = \mathbf{1}$: $\tilde{\mathbf{V}}_{qq} = \mathbf{V}_{qq}$, position and opacity couple through the shared precision matrix $\mathbf{V}_{qq}$ (similar to N-DGS, but with direct $\mathbf{V}_{pq}$ instead of derived $\boldsymbol{\Sigma}_{pq}$).
\item Intermediate values: Partial coupling with per-dimension control. Different query dimensions (e.g., view vs. time) can have different coupling strengths, with dimensions of the same type (e.g., the three view dimensions) sharing parameters.
\vspace{-0.5em}
\end{itemize}
This direct parameterization eliminates covariance inversion while providing fine-grained control over position conditioning strength. Each Gaussian learns optimal per-dimension coupling during training.

Inspired by N-DGS's design principle that displacement should scale with spatial extent (so small Gaussians produce small displacements), our displacement matrix $\mathbf{V}_{pq} = \bar{s} \boldsymbol{\Theta}_{pq}$ goes beyond their implicit covariance-based approach through direct parameterization. By normalizing $\boldsymbol{\Theta}_{pq}$ to unit Frobenius norm ($\|\boldsymbol{\Theta}_{pq}\|_F = 1$), displacements scale proportionally with the average Gaussian extent $\bar{s} = \text{mean}(s_x, s_y, s_z)$, with each row uniformly bounded by $\|\mathbf{V}_{pq}^{(i)}\|_2 \leq \bar{s}$. In contrast, N-DGS's implicit Cholesky structure yields a looser displacement bound of $\sqrt{C}(i{-}1 + s_i)$ per spatial row $i$, where the constant offset $(i{-}1)\sqrt{C}$ can dominate for small Gaussians and introduces asymmetry across spatial dimensions depending on dimension ordering. Our direct parameterization provides a tighter bound ($\bar{s}$) with no offset and uniform treatment across all spatial dimensions, better handling anisotropic Gaussians. (See Appendix~\ref{sec:spatial_scaling_proof} for the full derivation.)


\begin{algorithm}[tb]
\caption{\textit{d}GS Conditional Gaussian Slicing}
\label{alg:slice}
\begin{algorithmic}[1]
\REQUIRE Gaussian parameters: $\mu_p$, $\mu_q$, $\alpha$, $\mathbf{S}$, $\boldsymbol{\Theta}_{pq}$, $\mathbf{L}$, $\Lambda$
\REQUIRE Query point: $q$ (view/time)
\STATE $\boldsymbol{\delta} \gets q - \mu_q$ %\Comment{Compute query deviation}
\STATE $\alpha_{\text{cond}} \gets \alpha \cdot \exp(-\|\mathbf{L}^\top \boldsymbol{\delta}\|^2)$ % \Comment{Conditional opacity}
\IF{\textit{d}GS-O variant}
    \STATE \return  $\alpha_{\text{cond}}, \mu_p$ \Comment{Early exit for opacity only}
\ENDIF
\STATE $\mathbf{V}_{pq} \gets \text{mean}(\text{diag}(\mathbf{S})) \cdot \boldsymbol{\Theta}_{pq}$ % \Comment{Construct displacement matrix}
\STATE $\tilde{\mathbf{V}}_{qq} \gets \text{diag}(\boldsymbol{\Lambda}) \mathbf{L}\mathbf{L}^\top$ % \Comment{Scaled precision for coupling}
\STATE $\mu_{\text{cond}} \gets \mu_p + \mathbf{V}_{pq} \cdot \tilde{\mathbf{V}}_{qq} \cdot \boldsymbol{\delta}$ % \Comment{Conditional position}
\STATE \return $\alpha_{\text{cond}}$, $\mu_{\text{cond}}$
\end{algorithmic}
\end{algorithm}

\subsection{CUDA Implementation}

Since the original N-DGS methods, \ie, 6DGS \cite{gao20246dgs} and 7DGS \cite{gao20257dgs}, use PyTorch's autograd for conditional Gaussian operations without custom CUDA kernels, we implement both our disentangled formulation and the N-DGS baseline with optimized CUDA kernels for fair comparison. Our implementation uses a custom kernel that fuses conditional position and opacity computations in a single pass, as shown in Algorithm~\ref{alg:slice}.

\textbf{CUDA Kernel Design.} The kernel is templated on dimension $C$ (3 for 6D static scenes, 4 for 7D dynamic scenes) with compile-time loop unrolling for optimal performance. Each thread processes one Gaussian primitive independently, enabling perfect parallelization across millions of Gaussians. For opacity computation (Line 2 in Algorithm~\ref{alg:slice}), we compute the triangular matrix-vector product $\mathbf{L}^\top \boldsymbol{\delta}$ directly in registers using the lower-triangular structure—only $C(C+1)/2$ multiplications are needed, followed by a dot product for $\|\cdot\|_2^2$. For position displacement (Lines 6-8), we construct $\mathbf{V}_{pq}$ via diagonal scaling of $\tanh$ activations, then compute the scaled precision matrix $\tilde{\mathbf{V}}_{qq}$ by applying per-dimension scaling $\text{diag}(\Lambda)$ to $\mathbf{L}\mathbf{L}^\top$. All matrix operations use registers rather than shared memory, as the small matrix sizes fit entirely in per-thread registers. This fused design minimizes global memory bandwidth by avoiding intermediate tensor allocations and maximizes GPU occupancy through register-level computation.

\section{Experiments}

\subsection{Experimental Setup}

\textbf{Datasets.}
We evaluate \modelname{} across five diverse datasets covering both static and dynamic scenes:
\vspace{-1em}
\begin{itemize}[leftmargin=*]
    \item \textit{Static scenes:} (1) \textit{NeRF Synthetic}~\cite{mildenhall2021nerf} with 8 synthetic scenes under controlled lighting for evaluating geometric reconstruction; (2) \textit{Mip-NeRF 360}~\cite{barron2021mipnerf} with 9 challenging unbounded real-world scenes for evaluating view synthesis; and (3) \textit{6DGS-PBR}~\cite{gao20246dgs} featuring 7 physically-based scenes with complex view-dependent effects.

    \item \textit{Dynamic scenes:} (4) \textit{D-NeRF}~\cite{pumarola2020dnerfneuralradiancefields} with 8 synthetic sequences featuring various deformation types; and (5) \textit{7DGS-PBR}~\cite{gao20257dgs} containing 6 physically-based dynamic scenes with complex spatio-temporal-angular correlations.
\end{itemize}


\textbf{Evaluation Metrics.}
We assess reconstruction quality using Peak Signal-to-Noise Ratio (PSNR), Structural Similarity Index (SSIM)~\cite{wang2004image}, and Learned Perceptual Image Patch Similarity (LPIPS)~\cite{zhang2018perceptual}. We also report rendering speed (FPS), training time (minutes), and Gaussian count (K) to evaluate computational efficiency.

\textbf{Implementation Details.}
All methods are implemented in PyTorch on a single NVIDIA A100 GPU. We compare three configurations: 1) \textbf{N-DGS}: full N-DGS formulation with joint position and opacity conditioning, considered as our entangled baseline, 2) \textbf{\textit{d}GS-O}: our disentangled approach with only opacity conditioning ($\mu_{\text{cond}} = \mu_p$, no position displacement), and 3) \textbf{\textit{d}GS}: our full disentangled approach with both opacity conditioning and learnable position coupling via $\Lambda$. We use the same training settings with Adam optimizer for 30K iterations for all three configurations. For initialization, we set the view mean $\mu_d$ as random unit vectors (with time uniformly sampled in $[0, 1]$ for dynamic scenes), initialize displacement parameters $\boldsymbol{\Theta}_{pq}$ from $\mathcal{N}(0, 0.01^2)$, and set the Cholesky factor $\mathbf{L}$ with diagonal elements $\log(2.0)$ and off-diagonal elements 0.

\subsection{Results with MCMC Densification}

Table~\ref{tab:mcmc_comparison} compares our \modelname{} variants against N-DGS \cite{gao20246dgs} and 3DGS \cite{3dgs} baselines on static scene datasets with MCMC densification~\cite{kheradmand20243d}. Following the original paper settings, we set the maximum number of Gaussians to 300K for NeRF Synthetic scenes, and use scene-specific caps for Mip-NeRF 360 (ranging from 1.5M to 6M) and 6DGS-PBR (ranging from 60K to 300K) depending on scene complexity. Notably, our approach demonstrates significantly larger efficiency gains on large-scale scenes with millions of Gaussians. On Mip-NeRF 360 (1.5M-6M Gaussians), both \textit{d}GS-O and \textit{d}GS achieve substantial +166\% FPS speedups over N-DGS (200.17 vs 75.31 FPS and 200.56 vs 75.31 FPS, respectively), while improving quality (+0.48 dB and +0.79 dB PSNR). In contrast, on the smaller-scale 6DGS-PBR dataset (60K-300K Gaussians), speedups are more modest: +23\% for \textit{d}GS-O (588.23 vs 476.79 FPS) and +16\% for \textit{d}GS (551.02 vs 476.79 FPS), with \textit{d}GS improving quality by +0.28 dB. On NeRF Synthetic (300K Gaussians), both \textit{d}GS-O and \textit{d}GS outperform N-DGS in quality (+0.21 and +0.30 dB PSNR) while achieving significant speedups (+68\% and +70\% FPS).

\textbf{CUDA Implementation Benefits.} Comparing FPS vs FPS$_{\text{CUDA}}$ in Table~\ref{tab:mcmc_comparison} demonstrates the substantial performance gains from our optimized CUDA kernels. For N-DGS, CUDA implementation achieves 1.95$\times$ speedup on NeRF Synthetic (348.95 vs 179.12 FPS), 2.23$\times$ on Mip-NeRF 360 (75.31 vs 33.79 FPS), and 1.63$\times$ on 6DGS-PBR (476.79 vs 292.60 FPS). Our \textit{d}GS method shows even more dramatic improvements: 2.22$\times$ speedup on NeRF Synthetic (593.94 vs 267.33 FPS), 4.59$\times$ on Mip-NeRF 360 (200.56 vs 43.72 FPS), and 1.51$\times$ on 6DGS-PBR (551.02 vs 364.74 FPS). The larger speedups on Mip-NeRF 360 highlight how CUDA optimization becomes increasingly critical for large-scale scenes with millions of Gaussians, where our direct parameterization's elimination of matrix inversions maximally benefits from GPU parallelization.

Table \ref{tab:summary_results} summarizes quality metrics (PSNR, SSIM, LPIPS) and efficiency metrics (Gaussian count, training time, FPS) across all five datasets with standard 3DGS densification \citep{3dgs}, including both static and dynamic scenes. \textbf{Static Scenes.}
On NeRF Synthetic, both variants improve quality: \textit{d}GS-O by +0.46 dB and \textit{d}GS by +0.53 dB, with rendering speedups of 23\% and 15\% respectively. On Mip-NeRF 360, efficiency gains are most pronounced: \textit{d}GS-O achieves 89\% faster rendering with +0.78 dB improvement, while \textit{d}GS achieves 61\% faster rendering with +0.97 dB improvement. On 6DGS-PBR, \textit{d}GS-O shows modest 3\% speedup but -0.57 dB quality drop, while \textit{d}GS recovers and exceeds baseline quality (+0.64 dB).

Position conditioning proves essential for temporal dynamics. On D-NeRF, \textit{d}GS-O suffers -2.21 dB quality drop despite 11\% faster rendering, while \textit{d}GS with learnable coupling via $\Lambda$ exceeds N-DGS by +0.46 dB, demonstrating adaptive coupling surpasses fixed coupling. Importantly, the MCMC results in Table~\ref{tab:mcmc_comparison} cap all methods at the same Gaussian budget, and \textit{d}GS still outperforms N-DGS by +0.79 dB on Mip-NeRF 360, confirming that quality gains are from the parameterization, not additional primitives.

\subsection{Ablation Study}

To understand the contribution of each component, we conduct an ablation study comparing four configurations in Table~\ref{tab:ablation}: (1) \textbf{$\Lambda{=}0$} (\textit{d}GS-O): opacity conditioning only, (2) \textbf{$\Lambda{=}1$}: fixed full coupling, (3) \textbf{\textit{d}GS w/o $\bar{s}$}: learned $\Lambda$ without spatial scaling, and (4) \textbf{learned $\Lambda$} (\textit{d}GS): our full method.

\textbf{Learned $\Lambda$ outperforms both fixed extremes.} On NeRF Synthetic, learned $\Lambda$ achieves 33.91 dB vs 33.84 ($\Lambda{=}0$) and 33.78 ($\Lambda{=}1$). On 6DGS-PBR (view-dependent scenes), the gap widens: 37.75 dB vs 36.54 ($\Lambda{=}0$, +1.21 dB) and 37.31 ($\Lambda{=}1$, +0.44 dB). This confirms that optimal coupling is scene-dependent: on diffuse scenes $\Lambda{=}0$ slightly outperforms $\Lambda{=}1$, while on specular scenes $\Lambda{=}1$ is substantially better (+0.77 dB), validating that per-Gaussian adaptive coupling is the right design.

\textbf{Spatial scaling is critical.} Removing $\bar{s}$ from displacement degrades PSNR by 2.65 dB on NeRF Synthetic (33.91 vs 31.26) and 2.97 dB on 6DGS-PBR (37.75 vs 34.78). Position displacement without proper scaling (\textit{d}GS w/o $\bar{s}$) performs worse than no displacement at all ($\Lambda{=}0$), confirming that coupling displacement magnitude to Gaussian spatial extent is essential.

\section{Limitations and Future Work}

\textbf{Limitations.} Under standard densification, \modelname{} tends to retain more Gaussian primitives than N-DGS (e.g., 2463K vs 1273K on Mip-NeRF 360). While these additional primitives contribute positively to reconstruction quality and rendering remains faster due to per-primitive compute savings, they increase memory consumption. Under MCMC densification with equal Gaussian budgets, \modelname{} still outperforms N-DGS (e.g., +0.79 dB on Mip-NeRF 360), confirming that quality gains are not solely from additional primitives. Additionally, \modelname{} is currently evaluated only within the N-DGS Gaussian framework. Combining our direct parameterization with alternative kernels (e.g., Beta kernels in UBS~\citep{liu2025universal}) is a promising direction that could bring efficiency gains to a broader class of N-dimensional splatting methods. We present a concrete formulation for this combination (dBS: direct Beta Splatting) in Appendix~\ref{sec:dbs}, along with a direct comparison between \modelname{} and UBS on shared benchmarks.

\textbf{Future Work.} Our direct parameterization framework naturally extends to additional conditioning effects beyond opacity and position. Analogous to factorized attention mechanisms where Gaussian attributes (opacity, position) serve as \textbf{values} independently modulated by contextual variables (view direction, time) as \textbf{queries}, our approach provides systematic extensibility along two axes: the \textit{value space} (which Gaussian attributes can be conditioned, such as \textbf{rotation} or \textbf{color}) and the \textit{query space} (which variables drive conditioning, such as \textbf{lighting direction} or \textbf{material properties}). Each new conditioning effect can be directly parameterized with dedicated parameters and learnable coupling strength, avoiding the growing overhead in N-DGS where adding dimensions increases the cost of matrix inversions, regression multiplications, and covariance corrections. This modular design ensures per-Gaussian cost scales linearly with conditioning effects.

\section{Conclusion}

We presented \modelname{}, a direct parameterization for efficient N-dimensional Gaussian splatting. Our key insight is that conditional Gaussian formulas, while elegant, are computationally inefficient: each conditioning effect can be directly parameterized with dedicated parameters instead of being implicitly derived through covariance matrix. This direct approach delivers efficiency by eliminating matrix inversions and covariance corrections (5-6$\times$ faster slicing kernel), stability through explicit Cholesky parameterization, and flexibility via learnable per-dimension coupling strengths that control how strongly each effect responds to queries. Experiments demonstrate up to 2.66$\times$ faster rendering on in-the-wild scenes (Mip-NeRF 360) while matching or exceeding N-DGS quality across diverse benchmarks.

\section{Comparison to N-DGS Displacement Parameterization}
\label{sec:spatial_scaling_proof}

Both N-DGS \cite{gao20246dgs, gao20257dgs} and \modelname{} scale displacement with spatial scale $s_i$, but differ in how tightly this coupling is achieved and in their parameterization approach.

\textbf{N-DGS: Implicit coupling via Cholesky structure.} N-DGS parameterizes the full $(3+C) \times (3+C)$ covariance via Cholesky decomposition $\boldsymbol{\Sigma} = \mathbf{L}\mathbf{L}^\top$, where $\mathbf{L}$ is lower triangular with diagonal elements $L_{ii} = s_i$ (spatial and query scales) and off-diagonal elements $L_{ij} = \ell_{ij}$ for $i > j$, where $\ell_{ij} \in [-1,1]$ are bounded correlation parameters (initialized to zero).

To illustrate this structure, consider 6DGS with 3 spatial and 3 view dimensions ($C=3$). The Cholesky factor $\mathbf{L} \in \mathbb{R}^{6 \times 6}$ has the form:
\begin{equation}
\mathbf{L} = \left(\begin{array}{rrrrrr}
       s_1 &        0 &        0 &        0 &        0 &        0 \\
\ell_{21} &       s_2 &        0 &        0 &        0 &        0 \\
\ell_{31} & \ell_{32} &       s_3 &        0 &        0 &        0 \\
\ell_{41} & \ell_{42} & \ell_{43} &       s_4 &        0 &        0 \\
\ell_{51} & \ell_{52} & \ell_{53} & \ell_{54} &       s_5 &        0 \\
\ell_{61} & \ell_{62} & \ell_{63} & \ell_{64} & \ell_{65} &       s_6
\end{array}\right),
\end{equation}
where $s_1, s_2, s_3$ are spatial scales, $s_4, s_5, s_6$ are query dimension scales, and $\ell_{ij} \in [-1,1]$ are bounded correlations.

The displacement block $\boldsymbol{\Sigma}_{pq} = (\mathbf{L}\mathbf{L}^\top)_{1:3,4:6}$ emerges from matrix multiplication. Computing via row-column dot products yields:
\begin{equation}
\boldsymbol{\Sigma}_{pq} = \left(\begin{array}{rrr}
                           s_1 \ell_{41} &                            s_1 \ell_{51} &                            s_1 \ell_{61} \\
     \ell_{21}\ell_{41} + s_2 \ell_{42} &      \ell_{21}\ell_{51} + s_2 \ell_{52} &      \ell_{21}\ell_{61} + s_2 \ell_{62} \\
\ell_{31}\ell_{41} + \ell_{32}\ell_{42} + s_3 \ell_{43} & \ell_{31}\ell_{51} + \ell_{32}\ell_{52} + s_3 \ell_{53} & \ell_{31}\ell_{61} + \ell_{32}\ell_{62} + s_3 \ell_{63}
\end{array}\right).
\end{equation}
Note that $\boldsymbol{\Sigma}_{pq}$ contains only spatial scales $s_1, s_2, s_3$ and bounded correlations $\ell_{ij}$ -- the query dimension scales $s_4, s_5, s_6$ do \textit{not} appear. This occurs because displacement describes how spatial position changes with query deviation, which should scale with the Gaussian's \textit{spatial} extent, not its query dimension spread.

In general, each element of $\boldsymbol{\Sigma}_{pq}$ can be expressed as:
\begin{equation}
[\boldsymbol{\Sigma}_{pq}]_{ij} = \sum_{k=1}^{i} L_{ik} L_{(3+j),k} = \sum_{k=1}^{i-1} \ell_{ik} \ell_{(3+j),k} + s_i \cdot \ell_{(3+j),i}.
\end{equation}

Cross-terms from spatial correlations $\ell_{ik}$ (where $k < i$) accumulate in the displacement. The displacement in row $i$ satisfies:
\begin{equation}
\|\boldsymbol{\Sigma}_{pq}^{(i)}\|_2 \leq \sqrt{C}(i-1 + s_i).
\end{equation}
For large $s_i$, this approximates $s_i \sqrt{C}$, but the constant offset $(i-1)\sqrt{C}$ can dominate for small Gaussians. This bound also introduces an asymmetry across spatial dimensions: even when $s_1 = s_2 = s_3$, row $i$ accumulates more cross-terms than row 1, allowing arbitrarily different displacement magnitudes depending on the spatial dimension ordering. While N-DGS uses zero initialization of $\ell_{ij}$ to ensure displacement initially scales with $s_i$, training can learn non-zero correlations that introduce both the offset and this asymmetry.

\textbf{Our method: Explicit coupling via scalar scaling.} We directly parameterize $\mathbf{V}_{pq} = \bar{s} \boldsymbol{\Theta}_{pq}$, where $\bar{s} = \text{mean}(s_x, s_y, s_z)$ is the average spatial scale and $\boldsymbol{\Theta}_{pq} \in \mathbb{R}^{3 \times C}$ are learnable parameters normalized to unit Frobenius norm ($\|\boldsymbol{\Theta}_{pq}\|_F = 1$). This gives:
\begin{equation}
\|\mathbf{V}_{pq}^{(i)}\|_2 = \bar{s} \|\boldsymbol{\Theta}_{i,:}\|_2 \leq \bar{s}.
\end{equation}
Each element in row $i$ scales uniformly with $\bar{s}$ with no cross-terms from other spatial dimensions, ensuring displacement scales with the average Gaussian size even for tiny Gaussians. The unit Frobenius norm constraint ensures bounded displacements while allowing flexible direction learning.

\textbf{Key difference.} Both methods bound displacement by spatial scale, but differ fundamentally:
\begin{itemize}[leftmargin=*]
\item \textbf{N-DGS}: Displacement emerges \textit{implicitly} from Cholesky structure with bound $\sqrt{C}(i-1 + s_i)$. Cross-terms can allow disproportionately large displacements for small Gaussians and introduce asymmetry across spatial dimensions. For highly anisotropic Gaussians (e.g., $s_x \gg s_y, s_z$), displacement along each axis $i$ is constrained by its individual scale $s_i$, allowing excessive motion along large axes while limiting movement along smaller dimensions.
\item \textbf{\modelname{}}: Displacement is \textit{explicitly} parameterized with tight bound $\bar{s}$, ensuring proper scaling across all Gaussian sizes with uniform treatment of all spatial dimensions. Critically, using the mean $\bar{s}$ enables balanced displacement control: it allows sufficient movement along smaller dimensions while preventing excessive motion along larger axes, ensuring the Gaussian can move uniformly in all directions regardless of anisotropy.
\end{itemize}

Empirically, removing spatial scaling (ablation: \textit{d}GS w/o $\bar{s}$) degrades PSNR by 2.59 dB on NeRF Synthetic (Table~\ref{tab:ablation}), confirming that displacement scaling is critical. Our direct parameterization provides tighter bounds, avoids dimension-ordering artifacts, and enables learnable per-dimension coupling strength control via $\Lambda$.

\subsection{dBS: Direct Beta Splatting}

We propose \textbf{dBS} (direct Beta Splatting), which combines \modelname{}'s direct conditioning with UBS's Beta kernel. The key idea is to replace UBS's covariance-derived conditioning ($\boldsymbol{\Sigma}_{qq}^{-1}$-based) with \modelname{}'s direct parameterization while keeping the per-dimension Beta bandwidth control.

\textbf{UBS} (current formulation):
\begin{align}
d_i &= \tanh\!\left((\hat{\mathbf{L}}_q^{-1} \boldsymbol{\delta})_i^2 / 2\right), \\
\alpha_{\text{cond}} &= \alpha \prod_{i=1}^{C} (1 - d_i)^{4\beta_i}, \\
\boldsymbol{\mu}_{\text{cond}} &= \boldsymbol{\mu}_p + \boldsymbol{\Sigma}_{pq} \boldsymbol{\Sigma}_{qq}^{-1} \operatorname{diag}(\boldsymbol{\beta}_q) \boldsymbol{\delta}, \\
\boldsymbol{\Sigma}_{\text{cond}} &= \boldsymbol{\Sigma}_{pp} - \boldsymbol{\Sigma}_{pq} \boldsymbol{\Sigma}_{qq}^{-1} \operatorname{diag}(\boldsymbol{\beta}_q) \boldsymbol{\Sigma}_{qp}.
\end{align}

\textbf{dBS} replaces covariance-derived conditioning with \modelname{}'s direct parameterization:
\begin{align}
d_i &= \tanh\!\left((\mathbf{L}^\top \boldsymbol{\delta})_i^2 / 2\right), \\
\alpha_{\text{cond}} &= \alpha \prod_{i=1}^{C} (1 - d_i)^{4\beta_i}, \\
\boldsymbol{\mu}_{\text{cond}} &= \boldsymbol{\mu}_p + \mathbf{V}_{pq} \cdot \operatorname{diag}(\boldsymbol{\beta}_q) \cdot \mathbf{V}_{qq} \cdot \boldsymbol{\delta}, \\
\boldsymbol{\Sigma}_{\text{cond}} &= \boldsymbol{\Sigma}_{pp} \quad \text{(no correction)}.
\end{align}

Key differences from UBS:
\begin{itemize}[leftmargin=*]
\item \textbf{No inversion}: $\mathbf{L}$ is directly parameterized as a Cholesky precision factor, replacing $\hat{\mathbf{L}}_q^{-1}$.
\item \textbf{Direct displacement}: $\mathbf{V}_{pq}$ replaces $\boldsymbol{\Sigma}_{pq}\boldsymbol{\Sigma}_{qq}^{-1}$, eliminating the regression matrix.
\item \textbf{Unified coupling}: Per-dimension $\boldsymbol{\beta}_q$ from UBS replaces \modelname{}'s $\boldsymbol{\Lambda}$, controlling both opacity bandwidth and position coupling in a single parameter per dimension.
\item \textbf{No covariance correction}: As validated in our experiments, direct $\boldsymbol{\Sigma}_{pp}$ suffices.
\end{itemize}

dBS combines UBS's per-dimension Beta bandwidth control (enabling different bandwidths for diffuse vs specular, static vs dynamic) with \modelname{}'s efficient direct conditioning (5--6$\times$ faster slicing). Implementation and evaluation of dBS is a concrete next step for this line of work.
